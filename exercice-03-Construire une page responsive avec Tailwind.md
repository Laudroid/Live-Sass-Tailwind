### **TP : Construction d'une Landing Page Responsive avec Tailwind CSS**

**Objectif du TP :** Construire une page responsive avec Tailwind CSS.

**Énoncé du TP :** Réaliser une landing page responsive complète en utilisant exclusivement Tailwind CSS pour le stylisme. La page devra être fonctionnelle et visuellement attrayante sur différentes tailles d'écran, simulant la présentation d'un produit ou service fictif.

---

#### **Contexte**

La maîtrise de frameworks CSS utilitaires comme Tailwind est une compétence clé pour développer rapidement des interfaces utilisateur adaptatives. Ce TP vous permet de mettre en pratique les concepts fondamentaux de Tailwind CSS pour créer une interface moderne et réactive, essentielle dans le web d'aujourd'hui.

#### **Consignes Générales**

1.  **Environnement de Travail :**
    *   Utilisez un projet HTML/CSS/JS simple.
    *   L'intégration de Tailwind CSS via le CLI (recommandé) ou PostCSS est attendue. Assurez-vous que votre configuration permet la compilation des classes Tailwind.

2.  **Structure du Projet :**
    *   Organisez votre code de manière logique (ex: `index.html`, `src/input.css`, `dist/output.css` pour la sortie compilée de Tailwind).

3.  **Utilisation de l'IA :**
    *   L'IA est un outil puissant et un allié dans votre apprentissage. N'hésitez pas à l'utiliser pour :
        *   Générer des idées de contenu textuel (titres, descriptions, slogans pour votre produit/service fictif).
        *   Proposer des palettes de couleurs (que vous adapterez ensuite avec les classes Tailwind ou en configurant votre `tailwind.config.js`).
        *   **Suggérer des combinaisons de classes Tailwind pour des composants spécifiques** (ex: "Donne-moi les classes Tailwind pour un bouton primaire responsive", "Comment centrer un div verticalement et horizontalement avec Tailwind ?").
    *   **Important :** Ne vous contentez pas de copier-coller. Analysez les suggestions de l'IA, comprenez pourquoi elle propose ces classes, et adaptez-les si nécessaire. Votre capacité à expliquer vos choix et à justifier l'utilisation de certaines classes (même si l'IA les a suggérées) sera valorisée. C'est en comprenant que vous apprenez réellement.

4.  **Versionnement :**
    *   Utilisez Git pour suivre vos modifications. Des commits réguliers avec des messages clairs sont appréciés.

#### **Étapes de Réalisation**

Votre landing page devra inclure les sections suivantes, toutes stylisées avec Tailwind CSS et entièrement responsives :

1.  **Initialisation du Projet :**
    *   Mettez en place votre environnement Tailwind CSS (installation, configuration `tailwind.config.js`, `input.css`).
    *   Créez le fichier `index.html` de base.

2.  **Le Header (En-tête) :**
    *   Un logo (texte ou image).
    *   Une barre de navigation avec au moins 3 liens.
    *   Un bouton d'appel à l'action (ex: "Se connecter", "Essai gratuit").
    *   La navigation doit être responsive (ex: menu hamburger sur mobile, navigation complète sur desktop).

3.  **La Section Héroïque (Hero Section) :**
    *   Un titre accrocheur et une courte description du produit/service.
    *   Un ou deux boutons d'appel à l'action principaux.
    *   Une image ou une illustration pertinente (peut être une image de placeholder).
    *   Le contenu doit être bien agencé et lisible sur toutes les tailles d'écran.

4.  **La Section Fonctionnalités/Services :**
    *   Présentez au moins 3 fonctionnalités ou services clés de votre produit.
    *   Chaque fonctionnalité doit avoir un titre, une courte description et une icône (utilisez une bibliothèque d'icônes comme Heroicons, Font Awesome, ou des SVG simples).
    *   Organisez ces éléments dans une grille responsive (ex: 1 colonne sur mobile, 2 ou 3 colonnes sur desktop).

5.  **La Section Appel à l'Action (CTA) :**
    *   Une section dédiée pour inciter l'utilisateur à effectuer une action finale (ex: "Commencez dès aujourd'hui", "Téléchargez l'application").
    *   Un titre fort et un bouton CTA bien visible.

6.  **Le Footer (Pied de page) :**
    *   Liens de navigation secondaires (ex: "À propos", "Contact", "Confidentialité").
    *   Informations de copyright.
    *   Optionnellement, des liens vers les réseaux sociaux.

7.  **Affinage et Responsivité Globale :**
    *   Testez votre page sur différentes tailles d'écran (utilisez les outils de développement de votre navigateur).
    *   Assurez-vous que le design reste cohérent et fonctionnel à chaque breakpoint (`sm`, `md`, `lg`, `xl`).
    *   Portez une attention particulière aux espacements, aux tailles de texte et à l'alignement.

#### **Critères d'Évaluation**

*   **Conformité à l'énoncé :** Présence de toutes les sections demandées et utilisation exclusive de Tailwind CSS.
*   **Responsivité :** La page s'adapte correctement et harmonieusement aux différentes tailles d'écran.
*   **Qualité du code Tailwind :** Utilisation pertinente et efficace des classes Tailwind. Évitez le CSS personnalisé inutile.
*   **Esthétique et UX :** Design agréable, lisibilité, navigation intuitive et cohérence visuelle.
*   **Explication des choix :** Capacité à justifier l'utilisation de certaines classes ou approches, notamment celles suggérées par l'IA.
